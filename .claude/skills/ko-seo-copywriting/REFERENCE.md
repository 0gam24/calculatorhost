# 한국어 SEO 카피라이팅 레퍼런스

> **대상**: calculatorhost.com (YMYL 금융 계산기)
> **용도**: content-writer 에이전트 참조

---

## §1. 한국어 SEO 카피의 원칙

### 1-1. 검색 의도(Search Intent) 우선
- **정보성 검색** ("연봉 실수령액"): 명확한 답 + 자동화 도구 제공
- **상업성 검색** ("대출 한도"): 계산 + 상품 비교 + 신청 경로
- **거래성 검색** ("취득세 신고"): 계산 + 법령 원문 + 신고 경로

### 1-2. 한국어 특성 반영
- 어미 변화 고려: "계산", "계산하기", "계산법", "계산기"는 다른 키워드
- 한자어 + 순우리말 혼용: "소득" vs "수입", "공제" vs "뺌"
- 공공문서 어투와 친근한 어투 균형

### 1-3. 자연스러운 키워드 배치
- 1차 키워드: H1, 첫 문단, URL, title, description
- 2차 키워드: H2, 본문 자연스러운 문맥
- 롱테일: FAQ, 소제목
- **스터핑 금지**: 의미상 반복이면 OK, 문법 파괴 반복 NG

## §2. 계산기 페이지 본문 표준 구조

총 1500-2500자 (계산기 UI 제외)

### 2-1. H1 (페이지 제목)
```
{계산기명} 2026 | 최신 세율 반영 간편 계산기
```
예: "연봉 실수령액 계산기 2026 | 4대보험·소득세 자동 계산"

### 2-2. 리드 문단 (50-100자)
**누가·왜·무엇** 명확히.
```
2026년 최신 {세율/요율}을 반영한 {대상} {주제} 계산기입니다. 
{타깃 사용자}가 {목적}할 때 필요한 {결과물}을 1분 만에 확인할 수 있습니다.
```

### 2-3. [계산기 UI] (컴포넌트)

### 2-4. 결과 해석 섹션 (150-250자)
- 결과가 의미하는 바
- 평균 대비 위치 (가능한 경우)
- 다음 액션 힌트

### 2-5. "{주제}란 무엇인가" 섹션 (200-300자)
- 공식 정의
- 법조항 번호 인용
- 일반인이 이해할 비유

### 2-6. 계산 공식 단계별 설명 (300-500자)
```

  과세표준 = 총급여 - 근로소득공제 - 인적공제 ...
  (예시: 연봉 5000만 원 → 과세표준 2,450만 원)

2단계: [세율 적용]
  ...

3단계: [공제 적용]
  ...
```

### 2-7. 주의사항 / 실수 포인트 (200-300자)
- 흔한 오해
- 실제 사례

### 2-8. 절세/활용 팁 (200-300자)
- 합법적 절세 방법
- 놓치기 쉬운 공제

### 2-9. FAQ (§3 참조, 400-600자)

### 2-10. 관련 계산기 (링크 5개, 한 줄씩)

### 2-11. 업데이트 로그
```
- 2026-04-24: 2026년 1월 개정 세율 반영
- 2026-01-15: 자녀세액공제 조정
```

### 2-12. 면책 / 출처 (고정 블록)
```
본 계산기는 참고용 도구이며 법적 효력이 없습니다.

출처:
```

## §3. FAQ 작성 원칙

### 3-1. 질문 선정
- **Google People Also Ask** 수집
- **네이버 지식인** 핫이슈
- 고객 상담 반복 질문

### 3-2. 질문 포맷
자연스러운 구어체로:
- ❌ "소득세율은?"
- ✅ "연봉 5000만원의 소득세는 얼마인가요?"

### 3-3. 답변 포맷
- 100-200자
- **핵심 답**을 첫 문장에
- 근거/조건 이후 추가
- 필요 시 내부 링크

### 3-4. 예시
```markdown
Q: 연봉 5000만 원의 월 실수령액은 얼마인가요?

A: 부양가족 1인, 비과세 없음 기준으로 월 실수령액은 약 350만 원입니다. 
세전 월급 약 416만 원에서 4대보험(약 37만 원)과 소득세·지방소득세(약 29만 원)가 공제됩니다. 
자녀 수·비과세 수당 유무에 따라 10만 원 내외 차이가 날 수 있으므로 
[연봉 계산기](/계산기/연봉실수령액)에서 본인 조건으로 확인하세요.
```

### 3-5. 구조화 데이터
모든 FAQ는 `FAQPage` JSON-LD 마크업 (seo-auditor 협업).

## §5. 가이드 블로그 (MDX, 2000-3000자)

### 5-1. 제목 패턴
- "{문제 상황} 절세 방법 - 2026년 {제도명} 완전 가이드"
- 예: "일시적 2주택 양도세 비과세 3년 요건 완전 정리 (2026 최신)"

### 5-2. 구조
1. 들어가는 말 (사용자 상황 공감, 100-200자)
2. 핵심 요약 (bullet 3-5개)
3. 상세 설명 (600-1000자, 실제 사례 포함)
4. 자주 묻는 질문 (200-300자)
5. 계산기 링크 (CTA)
6. 관련 가이드

### 5-3. E-E-A-T 강화
- 실제 사례 ("서울 강남구 84㎡ 아파트...")
- 법조항 명시 ("시행령 §154의2")
- 공식 링크
- 저자/감수자 표시 (가능 시)

## §6. 메타태그 한국어

### 6-1. title
- 55-60자 (한글 기준 좀 더 여유)
- 키워드 앞쪽
- 연도 포함 (최신성)
- 브랜드명 끝에
```
연봉 실수령액 계산기 2026 | 4대보험 자동 | calculatorhost
```

### 6-2. description
- 150-155자
- 혜택/CTA 포함
- 자연스러운 문장 (광고 투 자제)
```
2026년 최신 소득세율과 4대보험 요율로 연봉 실수령액을 즉시 계산. 
자녀·부양가족·비과세까지 반영한 월 실수령과 시급 한눈에 확인. 완전 무료, 회원가입 불필요.
```

### 6-3. Open Graph
```
og:title = title (줄여서 가능)
og:description = description
og:image = /og/{slug}.png (1200x630)
og:type = website
og:locale = ko_KR
```

## §8. 맞춤법 / 스타일

### 자주 틀리는 것
- "이자율" (O) / "이자률" (X)
- "세율" (O) / "세률" (X)
- "%" 앞 띄어쓰기 없음 (4.5%)
- "원" 앞 띄어쓰기 없음 (5000만원 → 5000만 원이 표준어지만 실무는 붙여씀)
- **본 프로젝트 규칙**: 숫자 + 단위는 붙여 쓰기 (5000만원, 3억원)

### 수치 표기
- 천 단위 쉼표 필수 (1,234,567)
- 단위 혼용 지양 ("12억 5,000만" 보다 "12억 5천만" 또는 "1,250,000,000원")
- 한국 독자는 "억·천만" 단위 친숙

### 존댓말
- 본문: "~합니다" 기본
- 설명: "~입니다" 활용
- 친근함: "~해요" 적정 수준
- **본 프로젝트 톤**: 공손하되 전문성 유지 (공무원 문체 회피, 블로거 문체 회피)

## §9. 구조화 데이터 연계

content-writer는 작성만 / 구조화 데이터는 seo-auditor가 처리:
- `SoftwareApplication` (계산기 페이지)
- `FAQPage` (FAQ 섹션)
- `Article` (블로그 가이드)
- `HowTo` (사용 방법)
- `DefinedTerm` (사전 용어)

content-writer는 FAQ 질문/답을 해당 JSON-LD에 넣기 좋게 구조화해서 넘김.
