# Google AdSense 정책 레퍼런스

> **출처**: https://support.google.com/adsense/answer/48182 외 공식 문서 (2026-04 기준)
> **대상**: calculatorhost.com
> **용도**: adsense-guardian 에이전트 참조용 진실 공급원

---

## §1. 금지 콘텐츠 유형

AdSense는 다음 콘텐츠에 광고 게재 금지 (발견 시 계정 정지 가능):
- 성인물·노출물·성적 콘텐츠
- 폭력·잔혹·테러 옹호
- 증오/차별 콘텐츠
- 저작권 침해 (불법 다운로드/스트리밍)
- 불법 약물·무기·알코올·담배 판촉
- 도박·베팅 (일부 지역 제한적 허용)
- 해킹/크래킹 도구
- 사기·피싱
- 비방·명예훼손

**본 프로젝트 상태**: 계산기/세금/금융 도구 → 대부분 안전.
**주의**: 
- "투자 권유"로 해석될 수 있는 문구 금지 ("이 주식 사세요" X → "계산 결과" O)
- 대출 상품 "모집"처럼 보이는 CTA 금지

---

## §2. 무효 트래픽 (Invalid Traffic) — 계정 정지 1순위

**금지 행위**:
- **자기 클릭 금지** (본인 사이트 광고 클릭 절대 금지)
- 반복적 수동 클릭, 자동화 도구, 봇·크롤러 사용
- 클릭 교환 프로그램 참여
- 유료 방문자 구매
- 인센티브 제공 ("클릭하면 경품" 등)
- 가족/친구/직원에게 클릭 요청

**본 프로젝트 규칙**:
- 개발/테스트 시 광고 로드는 하되 **클릭 절대 금지**
- 로컬 개발 환경은 ads.txt 제외 또는 테스트 광고 사용
- 팀원·친구에게 "사이트 방문해줘" 요청 시 반드시 "광고 클릭 금지" 고지

---

## §3. 허용되는 광고 배치

- 주요 콘텐츠 상단/사이드바/하단
- 콘텐츠 사이 (inline, in-feed)
- 기사/포스트 중간
- 사이트의 **가치 있는 콘텐츠가 주된** 페이지

**본 프로젝트 권장 배치**:
- 우측 사이드바 (knowingasset.com 방식 — 데스크톱 전용)
- 계산 결과 카드 하단 (인피드)
- 설명 콘텐츠 중간 (1500자 이상 글에서 2곳 이하)
- 모바일 하단 고정 (주의: 콘텐츠 가림 금지)

---

## §4. 금지된 광고 배치

절대 광고 넣으면 안 되는 위치:
- **소프트웨어 앱 / 툴바 / 브라우저 확장**
- **팝업 / 팝언더 창**
- **이메일 / DM / 개인 메신저**
- **콘텐츠 기반이 아닌 페이지** (404, 로그인, 결제, 빈 페이지)
- **자동 리다이렉트 페이지**
- **다른 사이트를 프레이밍(iframe)한 페이지**
- **광고 목적만 있는 페이지** (Made For AdSense, MFA)
- **네비게이션으로 오인될 위치** (메뉴처럼 생긴 광고)
- **동의 없이 자동 재생되는 미디어 옆**
- **전면 광고(Interstitial)로 모바일 콘텐츠 가림** → 구글 SEO 순위도 동반 하락

**본 프로젝트 차단 위치**:
- /privacy, /terms, /contact, /about (정책 페이지에도 광고 X)
- 404 페이지
- 검색 결과 0건 페이지

---

## §5. 광고 밀도 & 사용자 경험

공식 수치 상한은 없으나, 사용자 경험 정책에 따라:
- 광고가 **주요 콘텐츠를 압도하면 안 됨**
- "광고 밀도가 콘텐츠보다 높은 페이지" = 정책 위반 위험
- 모바일에서 스크롤 없이 광고만 보이는 상태 금지

**본 프로젝트 상한**:
- 페이지당 디스플레이 광고 **≤ 4개**
- 자동 광고(Auto Ads) 사용 시 `data-ad-frequency-hint="30s"` 이상
- 접힌 영역(above the fold)에 광고 ≤ 2개
- 모바일에서 첫 화면 콘텐츠 대비 광고 비율 ≤ 30%

---

## §6. 광고 라벨링

**허용되는 라벨**:
- "스폰서 링크" / "Sponsored Links"
- "광고" / "Advertisement"

**금지되는 라벨** (오도 소지):
- "즐겨찾는 사이트"
- "추천 상품"
- "오늘의 인기"
- "관련 계산기" (← 우리 내부 콘텐츠 섹션 이름과 충돌 주의!)

---

## §7. 클릭 유도 금지

광고에 대해 다음 언급/유도 금지:
- "광고 클릭하세요" / "Click here"
- "이 광고 확인해보세요"
- "도와주세요" (암시적 구걸)
- 화살표·강조 테두리로 광고 가리키기
- 광고 옆 오해 유발 이미지 (예: "무료 쿠폰!" 옆 광고)
- 광고와 콘텐츠를 구분 불가능하게 디자인

---

## §8. ads.txt 필수 (2017+ 기본)

- 루트 도메인 `/ads.txt` 파일 필수
- Google AdSense 퍼블리셔 ID 포함:
  ```
  google.com, pub-0000000000000000, DIRECT, f08c47fec0942fa0
  ```
- Cloudflare Pages에서는 `public/ads.txt` 로 배포
- Search Console과 AdSense 대시보드에서 검증

---

## §9. app-ads.txt (앱 없음 → 스킵)

본 프로젝트는 웹 전용 → 불필요.

---

## §10. 개인정보·쿠키 공개 (EU/UK GDPR, CCPA 대응)

- **privacy-policy 페이지에 명시 필수**:
  - Google/AdSense 쿠키 사용
  - 관심사 기반 광고 (DART 쿠키)
  - 제3자 광고 제공자 링크 (https://policies.google.com/technologies/ads)
  - 사용자 옵트아웃 경로
- EEA/영국/캘리포니아 사용자에게 **동의 배너** 필수 (AdSense가 자동 처리 옵션 제공)
- 계산기 사이트도 **개인정보처리방침 페이지는 절대 필수**

**본 프로젝트 약관 페이지 필수 요소**:
- [ ] AdSense 사용 공개
- [ ] 쿠키 정책
- [ ] 제3자 추적 기술 명시
- [ ] 사용자 옵트아웃 방법
- [ ] 연락처

## §11. CLS 방지 — 광고가 SEO에 주는 영향

AdSense 광고는 Core Web Vitals CLS 악화 1순위:
- 광고 로드 전후 레이아웃 변경 → CLS ↑
- SEO 순위 하락 직결

**본 프로젝트 필수 조치**:
- 모든 광고 슬롯 **고정 크기 예약** (`min-height: 250px;` 등)
- 반응형 광고도 `max-height` 설정
- `<ins class="adsbygoogle">` 부모 요소에 명시적 크기
- `next/script strategy="lazyOnload"` 적용 (LCP 보호)
- Loading Skeleton으로 로드 전 placeholder 표시

---

## §12. 자동 광고(Auto Ads) 사용 시 주의

- 편리하나 CWV 악화 위험
- 광고 밀도 제어 어려움
- 설정에서 `data-ad-frequency-hint` 로 조절
- 본 프로젝트 초기: **수동 배치 권장** (광고 5-6개로 시작 → 데이터 보고 튜닝)

---

## §13. 제재 단계

1. **1단계**: 특정 사이트 광고 게재 중단
2. **2단계**: 계정 사용 중지 (경고 단계)
3. **3단계**: 계정 영구 해지 → AdSense 프로그램 영구 참여 불가

**영구 차단 사유 (특히 주의)**:
- 반복적 자기 클릭
- 의도적 정책 위반
- 계정 정지 후 우회 시도 (친구/가족 명의)

---

## §14. YMYL + AdSense 특별 고려

금융·세금 계산기는 YMYL → AdSense도 "사용자 신뢰" 엄격 요구:
- 광고와 계산 결과의 시각적 구분 명확 (색/간격/라벨)
- 계산 결과 옆에 "아래는 광고입니다" 명시 권장
- 대출/보험 광고 표시 시 계산기 결과와 혼동 없게

---

## §15. 배포 전 AdSense 체크리스트

### 필수 (배포 차단)
- [ ] AdSense 계정 승인됨 (도메인 수준)
- [ ] `ads.txt` 루트에 배포
- [ ] `privacy-policy`에 AdSense/쿠키 공개
- [ ] `terms` / `contact` 페이지 존재
- [ ] 광고가 콘텐츠로 오인되지 않음
- [ ] 광고 라벨 "광고" 또는 "Sponsored" 표시
- [ ] 팝업/자동재생 없음
- [ ] 로컬 개발 시 자기 클릭 방지 (ads.txt 제외 or ENV 분기)

### 기술 (CWV 보호)
- [ ] 모든 광고 슬롯 고정 크기 예약
- [ ] `strategy="lazyOnload"` 또는 `afterInteractive`
- [ ] CLS < 0.1 검증
- [ ] LCP 영향 측정

### 권장
- [ ] 페이지당 광고 ≤ 4개
- [ ] EEA 동의 배너 (Funding Choices)
- [ ] 광고 위치 A/B 테스트 준비

---

## §16. 개발 중 안전 프로토콜

- 로컬 환경에서 실제 광고 클라이언트 ID 사용 금지
- `NEXT_PUBLIC_ADSENSE_CLIENT` 환경 변수로 prod/dev 분리
- dev 환경: `ca-pub-0000...` (가짜 ID) 또는 광고 스크립트 비활성
- 스테이징에서 테스트 시: "나·팀원 광고 클릭 금지" 팀 규칙 공지
- AdSense 정책 업데이트 알림 구독 (support.google.com/adsense)

---

## §17. 위반 발견 시 대응 순서

1. 즉시 해당 광고/페이지 제거 또는 비활성
2. AdSense 정책 위반 검토 (`policy center`)
3. 개선 후 재검토 요청
4. 반복 위반 방지를 위해 CI에 자동 검사 추가 (adsense-guardian 에이전트 상시 실행)
