# AdSense 정책 레퍼런스 스킬

## 워크플로우

### 상황 1: 새 광고 슬롯 추가 요청
1. `REFERENCE.md` §3(허용 배치) / §4(금지 배치) 확인
2. 페이지당 광고 밀도 점검 (§5)
3. 슬롯 컴포넌트가 CLS 유발 안 하도록 크기 예약 확인
4. `strategy="lazyOnload"` 적용 여부 확인
5. 허용되면 승인, 아니면 대안 제시

### 상황 2: 기존 광고 배치 감사
1. 모든 광고 슬롯 위치 Grep → 목록화
2. `REFERENCE.md` §3-§6 체크리스트 전부 적용
3. 🔴🟡🟢 분류 + 수정 지시

### 상황 3: 콘텐츠 정책 위반 검사
1. `REFERENCE.md` §1(금지 콘텐츠) 기준
2. 계산기 주제가 YMYL 금융이라 대부분 안전하나, 콘텐츠에 투자 권유성 문구 있는지 확인
3. AI 생성 콘텐츠 공개 여부 확인

### 상황 4: 배포 전 최종 점검
1. ads.txt 파일 존재 확인
2. `privacy-policy` 페이지에 AdSense/쿠키 공개 문구 포함 확인
3. 광고 개수, 라벨링, 사용자 경험 최종 체크

## 통과 기준
- 페이지당 광고 슬롯 ≤ 4개
- 모든 광고 슬롯 `min-height` 예약 (CLS 방지)
- 자동 광고 사용 시 `data-ad-frequency-hint` 설정
- 개인정보처리방침에 AdSense/쿠키 문구 명시

## 에스컬레이션
- 계정 정지 사유 발견 시 즉시 사용자에게 에스컬레이션
- 정책 모호한 경우 Google 공식 문서 URL 제시
