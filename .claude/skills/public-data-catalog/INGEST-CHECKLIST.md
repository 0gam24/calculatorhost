## 체크리스트 템플릿

**사용법**: 새 포스트 작성 시 아래 목록을 읽고, 해당되는 토픽에 ✅ 체크 + API 매핑.

### A. 금리/환율 가이드 (대출·적금·환전)

| 체크항목 | 권장 API | 레벨 | 비고 |
|---|---|---|---|
| [ ] 현재 기준금리 인용 (월 대출 이자 자동 기본값) | ECOS 722Y001 | 필수 | `lib/publicapi/ecos.ts` → 1일 1회 cron |
| [ ] 은행별 정기예금/적금 평균금리 비교표 | FSS (finProdList.xml) | 필수 | 상단 Structured Summary 테이블 |
| [ ] 원달러 현시점 환율 (선택) | ECOS 731Y001 또는 EXIM API | 선택 | 환전 계산기만 |
| [ ] "금리가 올라/내려도 내 대출은?" 시뮬 링크 | 내부 대출이자 계산기 | 권장 | 대출 관련 가이드 모두 |

### B. 부동산 거래 가이드 (양도세·취득세·실거래가)

| 체크항목 | 권장 API | 레벨 | 비고 |
|---|---|---|---|
| [ ] 인근 지역 최근 실거래가 데이터 (3건 이상) | RTMS (§2-1, 아파트 매매) | 권장 | 지역별·단지별 사례 |
| [ ] 단지명 → 시세 자동 입력 (양도세 계산기 연동) | RTMS + 법정동코드 변환 | 권장 | 초기: 수동 검색, 추후 자동 |
| [ ] 전월세 실거래가 (전월세전환 가이드) | RTMS (§2-2, 아파트 전월세) | 권장 | 최근 갱신일자 명시 |
| [ ] 공시지가·공정가격 (종부세·재산세 기반) | KOSIS 지역별 주택가격 지수 | 선택 | "공시가격 어떻게 나올까?" 비교 |

### C. 소득/복지 가이드 (연봉·프리랜서·자녀)

| 체크항목 | 권장 API | 레벨 | 비고 |
|---|---|---|---|
| [ ] 가구 기준 중위소득 vs 본인 비율 비교 | KOSIS (가구소득·중위소득) | 권장 | "연봉 5000만은 상위 ?" |
| [ ] 경기도/서울/지역별 평균 가구소득 | KOSIS 시도별 분류 | 선택 | 광역시별 비교표 |
| [ ] 4대보험료 계산 기준액 + 인상률 | 국세청(수동) + KOSIS | 권장 | 연도별 인상률 공시 |
| [ ] 자녀장려금/교육비공제 통계 (모집단) | KOSIS 세금납부자별 통계 | 선택 | "얼마나 혜택 받을까?" 기댓값 |

### D. 투자/자산 가이드 (복리·물타기·인플레이션)

| 체크항목 | 권장 API | 레벨 | 비고 |
|---|---|---|---|
| [ ] 최근 5년 인플레이션율 (물가지수) | ECOS CPI (901Y009) | 권장 | 복리·시뮬레이션의 기초 |
| [ ] 역사적 평균 금리 (2020-2026) | ECOS 신규/잔액 COFIX + 기준금리 | 선택 | 장기 시뮬 시 비교기준선 |
| [ ] 주식/펀드 데이터 (자동주가 X, 수동만) | 없음 (민간 유료/비공식) | 선택 | 물타기 가이드는 예시만 |

### E. 법정동코드·주소 변환 (보조)

| 체크항목 | 권장 API | 레벨 | 비고 |
|---|---|---|---|
| [ ] "지역 선택" → 법정동코드 자동 맵핑 | JUSO (road2addr) 또는 정적 테이블 | 필수\* | \*계산기에만 필수, 가이드는 선택 |
| [ ] 지번 → 도로명 주소 변환 (선택) | JUSO (addr2road) | 선택 | 사용자 편의 |

## 포스트별 적용 예시

### 예시 1: "양도세 일시적 2주택" 4티어 가이드 (2026-05-13 발행)
```
✅ 체크항목:
- [ ] 인근 지역 최근 실거래가 데이터 (3건 이상) → RTMS (§2-1)
      → Structured Summary 테이블: 서울 강남구 2026년 4월 평균
- [ ] 단지명 → 시세 자동 입력 (양도세 계산기 연동) → FUTURE (아직 미연동)
- [ ] 공시지가·공정가격 → KOSIS (선택, 이 가이드는 미적용)

→ 1개 이상 API 적용 → E-E-A-T 강화 ✅
```

### 예시 2: "프리랜서 종합소득세 절세 5가지" 4티어 가이드 (FUTURE)
```
✅ 체크항목:
- [ ] 가구 기준 중위소득 vs 본인 비율 비교 → KOSIS
      → "연소득 3000만은 중위소득 하위 ?" 데이터 섹션
- [ ] 4대보험료 계산 기준액 + 인상률 → 국세청 + KOSIS
      → "2026 기준 국민연금 최대 납부액" 표

→ 2개 이상 API 적용 → 신뢰도 ⬆
```

### 예시 3: "전월세 전환율 vs 보증금 대출" 비교 가이드 (FUTURE)
```
✅ 체크항목:
- [ ] 전월세 실거래가 → RTMS (§2-2)
      → Structured Summary: "최근 6개월 서울 전월세 계약 건수"
- [ ] 기준금리 → ECOS 722Y001
      → "현재 대출금리는 기준금리 + ?" 비교
- [ ] 인근 지역 최근 실거래가 → RTMS (§2-1)

→ 3개 API, GEO/AEO 최상 ✅
```

## 배포 전 필수 확인

### 레벨 1: 데이터 신선도 (배포 -1일)
- [ ] API 응답 timestamp 확인 (24시간 이내 갱신?)
- [ ] 인라인 수치가 페이지 발행 기준일로부터 ±3일 이내 수집?

### 레벨 2: 인용 형식
- [ ] "2026-05-14 기준 한국은행 기준금리 X%" ← 출처 + 날짜 명시
- [ ] Structured Summary 테이블은 HTML `<table>` (이미지 X)
- [ ] 각 수치 옆 `[출처]` 마크다운 링크 (공식 페이지로)

### 레벨 3: 동적 vs 정적 판단
- [ ] 수치가 월 1회 이상 변동? → Edge Function/cron 캐시 필수
- [ ] 정적 한 번? → 본문에 고정 + 갱신일자 명시만

### 레벨 4: 법적 명시
- [ ] 공공데이터 라이선스 (CC-BY-4.0) 확인 → 저작권 면책 문구 필요?
  - 대부분 공공데이터포털 = 자유로운 이용 가능
  - "출처: 공공데이터포털" 한 줄이면 충분

## 주요 주의사항

1. **RTMS/JUSO 키 미확정** → 부동산 가이드 우선순위 낮추기 (대안: 법조항·계산기 링크만)
2. **FSS API 미구현** → 금리 가이드는 ECOS만 먼저 (적금·예금은 FUTURE)
3. **일일 쿼터 관리**:
   - ECOS 10,000건/일
   - KOSIS 별도 상한 확인 (보통 500-1000건/일)
   - 캐싱으로 중복 호출 방지
4. **Cloudflare Edge 주의**:
   - Node.js fs 모듈 X (정적 파일 읽기 불가)
   - fetch만 사용 (모든 API는 HTTP로)
5. **데이터 정확성**:
   - AI가 인용할 가능성 높음 (GEO/AEO)
   - 잘못된 수치 = 신뢰도 급락
   - 배포 전 샘플 케이스로 수기 검증
