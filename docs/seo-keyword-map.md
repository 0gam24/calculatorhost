# SEO 키워드 × 페이지 매핑

> **Status**: SSoT — content-writer·seo-auditor 참조
> **Last updated**: 2026-04-24

## 1. 목표 재정의

**"계산기" 단일 키워드 구글 1등**은 현실적 12-18개월 소요.
단기 목표는 **2-3티어 롱테일 20-50개 1등** 누적으로 브랜드화.

## 2. 키워드 4티어 전략

### 1티어 (월 검색량 10만+, 경쟁 극심)
접근 신중. 초기 집중 아님.
- 연봉 계산기
- 대출 계산기  
- 환율 계산기
- BMI 계산기

### 2티어 (월 5-10만, 현실적 1등 목표)
**주력 타깃**.
- 양도소득세 계산기
- 취득세 계산기
- 퇴직금 계산기
- DSR 계산기
- 중개수수료 계산기
- 평수 계산기
- 재산세 계산기
- 전세자금대출 한도
- 연봉실수령액 계산기

### 3티어 (월 1-5만, 블루오션)
**차별화 기회**.
- 1주택 양도세 비과세
- 일시적 2주택 양도세
- 청약가점 계산기
- 증여세 10년
- 종합부동산세 2026
- DC DB 퇴직금 비교
- 프리랜서 종합소득세
- 4대보험 계산기
- 자동차세 계산기
- N잡러 건강보험

### 4티어 (롱테일 300+)
**SEO 초반 승부처**.
각 계산기당 5-10개 FAQ·가이드 변형 목표.

예시:
- "연봉 5000 실수령액 2026"
- "취득세 8.4% 3주택 85제곱"
- "양도세 일시적2주택 3년"
- "전세자금대출 2억 월이자"
- "자녀 2명 소득공제"

## 9. GEO/AEO 원칙 (2026 필수)

**배경**: 구글 검색 60%+가 제로 클릭. LLM(ChatGPT/Gemini/AI Overview)이 답변을 합성. 우리의 생존 전략 = "LLM이 우리를 인용하게 만든다".

### 본문 구조 필수 요소 (모든 계산기 페이지)
1. **리드 문단 직후 Structured Summary**:
   - 정의 블록 (50-80자)
   - 핵심 수치 `<table>` (3-5행)
   - TL;DR 박스 (3-5줄)
2. **FAQ를 중간 이상 배치** (하단 아닌 본문 속)
3. **답변 첫 문장 = 결론** 원칙
4. **데이터는 HTML 테이블** (이미지/스크린샷 금지)

### 구조화 데이터 세트 (5종)
- SoftwareApplication (필수)
- FAQPage (필수)
- BreadcrumbList (필수)
- HowTo (사용 방법 섹션 있으면)
- Speakable (답변형 콘텐츠, 음성 최적화)
- DefinedTerm (용어사전 페이지만)

### 정보 공백(Opportunity Voids) 공략
content-writer 작업 패턴:
1. 타깃 키워드로 AI Overview 답변 확인
2. AI가 놓친 엣지 케이스 식별
3. 그 엣지 케이스를 중점으로 콘텐츠 작성
→ LLM이 향후 인용할 가능성 ↑

## 10. 브레이크아웃 키워드(Velocity) 전략

**배경**: Ahrefs/Semrush는 30-90일 전 데이터. 상위는 대형 미디어 선점. 신규 진입 불가.

**대안**: Google Trends 실시간 10분 단위 예측 엔진 활용 → 검색량 5000%+ 폭증 중인 '브레이크아웃' 키워드 초기 선점.

### 월례 프로세스 (seo-auditor 주관)
1. Google Trends "최근 24시간" 급상승 모니터링
2. 금융·세금·부동산 카테고리 필터
3. 지속적 상승(Active) vs 단발(Lasted) 필터링
4. 매칭되는 계산기/가이드 즉시 발행
   - 예: 세법 개정 뉴스 → 개정안 해설 가이드 즉일 발행
   - 예: 전세사기 이슈 급증 → 전세보증금 안전 가이드

### 자동화 후보 (Phase 2+)
velocity-trend-monitor 스킬/에이전트 검토 (월 1회 리포트 자동화)
