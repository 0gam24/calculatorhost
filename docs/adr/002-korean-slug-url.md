# ADR-002: 한글 슬러그 URL

**상태**: 수락됨  
**일자**: 2026-04-24

## 맥락
URL 설계 시 영문 슬러그 (`/calculator/salary`) vs 한글 슬러그 (`/계산기/연봉실수령액`) 선택 필요.

## 결정
**한글 슬러그 채택**.

예시:
- `/계산기/연봉실수령액`
- `/계산기/양도소득세`
- `/사전/과세표준`
- `/가이드/일시적2주택-양도세`

## 근거
1. **검색 의도 매칭 강함**: 한국 사용자가 검색하는 쿼리와 URL이 일치하면 CTR/순위 상승 효과
2. 부동산계산기.com 등 국내 성공 사례 다수
3. 한글 URL도 브라우저에서 자동 인코딩됨 (`%EA%B3%84%EC%82%B0%EA%B8%B0`)
4. Google이 한글 URL을 잘 처리함 (공식 문서에서 권장)

## 대안 고려
- **영문 슬러그**: URL 짧음, 공유 시 깔끔. 하지만 한글 검색 의도와 거리.
- **영문+한글 리다이렉트**: 복잡도 증가.

## 결과
- Next.js App Router 디렉토리명에 한글 사용
- 내부 구성은 특수문자 없는 순수 한글
- 공유 시 URL 인코딩됨 (카톡·이메일에서 디코딩되어 정상 표시)

## 리스크
- 일부 오래된 도구/라이브러리에서 인코딩 문제 → 빌드/배포 전 QA 필수
- 영문 호환성이 필요하면 `rewrite` 또는 중복 경로 검토
