# ADR 008: AdSense 발사 후 운영 플레이북 수립

**Date**: 2026-05-04  
**Status**: Accepted  
**Context**: Phase B (신청 직전 감사) 완료. Phase C (발사 후 운영)에 진입하기 위해 월간·분기 운영 규칙을 영구 문서화.

## 문제

AdSense 계정은 일단 활성화되면:
1. **정책 위반 발견 = 즉시 계정 정지** 위험 (YMYL 도메인)
2. **월간 모니터링 없음 = 무효 트래픽 누적** 가능
3. **새 계산기 추가 시마다 정책 검수** 필요 (자동화 부재)
4. **운영자 교체 시 인수인계 문서 없음**

→ 체계적 playbook 없으면 수익화 불안정.

## 결정

`docs/adsense-launch-playbook.md` 신규 작성. 구조:
- **Day 0 신청 체크리스트** (신청 전/후)
- **Day 1–7 거부 사유별 대응** (3가지 주요 사유)
- **Day 0 (승인 직후) 활성화** (ads.txt, ENV 변수, 정책 페이지)
- **Week 1 실시간 모니터링** (일일 점검)
- **Week 2–4 점진적 광고 노출** (5 슬롯 순서)
- **월간 운영 체크리스트** (수익 분석, 정책 업데이트)
- **분기/연간 검토** (성과 최적화)
- **위반 발견 시 에스컬레이션** (즉시→1시간→배포)
- **금기사항** (8가지 계정 정지 위험 행동)
- **Troubleshooting** (광고 안 보임, CLS, CTR)

### 근거
- `.claude/skills/adsense-policy-reference/REFERENCE.md` § 8, 10, 13–17
- Google AdSense 공식 정책: https://support.google.com/adsense/answer/48182
- Phase B 감사 완료 (신청 가능 상태 확인)

## 영향

### 긍정
- **계정 정지 예방**: 정책 위반 조기 발견 + 월간 모니터링
- **운영 자동화**: 슬롯 배치 타이밍, 성과 분석 체계화
- **팀 협업**: 신규 콘텐츠 추가 시 adsense-guardian 호출 프로세스 명시
- **신뢰성**: YMYL 도메인에 적합한 보수적 운영 기준

### 비용
- 월간 1시간 모니터링 소요 (자동화 전까지)
- 슬롯 추가 시 정책 검수 (adsense-guardian 호출)

## 대안 검토

1. **대안 A**: Playbook 미작성
   - 비용: 0
   - 위험: 정책 위반 반복 → 계정 정지 가능성 높음 ❌

2. **대안 B**: 일반 AdSense 가이드 참조만 (외부 링크)
   - 비용: 0
   - 한계: 본 프로젝트 특성 미반영 (YMYL, 5 슬롯, 월간 프로세스)
   - 선택: 부분 채택 (참고 자료로 외부 링크 포함)

3. **대안 C**: 전사 운영 매뉴얼 (Notion/Wiki)
   - 비용: 높음 (별도 플랫폼 관리)
   - 선택: 미연기 (현재는 문서 저장소로 git 사용)

**선택**: 대안 C (本 결정 — playbook 작성)
