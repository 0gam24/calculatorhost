# ADR-006: GEO/AEO를 SEO 최우선 원칙으로 격상

**상태**: 수락됨  
**일자**: 2026-04-24

## 맥락
2026년 현재 전 세계 구글 검색 60%+가 제로 클릭. 사용자가 AI 오버뷰·ChatGPT 요약만 보고 웹사이트 방문 안 함. 전통적 SEO는 트래픽 유입 구조적 한계 도달.

보고서 §5.2 해결책: **생성형 엔진 최적화(GEO) / 답변 엔진 최적화(AEO)** — LLM이 인용할 수 있도록 콘텐츠를 구조화. 인용되면 역설적으로 트래픽 유입.

## 결정
**모든 페이지 콘텐츠 구조를 "LLM 인용 가능성 극대화"를 제1원칙으로 설계.**

### 구체적 구현 규칙

#### 1. 본문 상단 Structured Summary (필수)
계산기 페이지 리드 문단 직후에:
- **정의 블록** (50-80자): "{주제}란 {정의}입니다."
- **핵심 수치 테이블** (3-5행): 가장 자주 묻는 케이스 요약표
- **TL;DR 박스**: 결론 요약 3-5줄

→ LLM이 이 부분을 인용하며 출처 링크 첨부할 확률 ↑

#### 2. Q&A 중심 콘텐츠 구조
- FAQ 섹션을 하단이 아니라 **중간 이상**으로 격상
- 각 FAQ 질문은 **자연어 대화형**
- 각 답변: 첫 문장 = 핵심 결론, 이후 근거
- People Also Ask 영역 타깃

#### 3. 데이터 테이블 전면 배치
- 세율표·공제표·금리표를 **스크린샷 아닌 `<table>` HTML**로
- `<caption>` + `<th>` 의미론적 태그 필수
- LLM이 데이터 추출하기 쉬운 구조

#### 4. 정보 공백(Opportunity Voids) 발굴
- content-writer 에이전트에 작업 D 추가: "Google PAA + AI 답변에 빠진 정보 공백 탐색"
- 경쟁자·AI 답변이 놓친 엣지 케이스를 집중 작성

#### 5. 구조화 데이터 확장
기존 SoftwareApplication + FAQPage + BreadcrumbList에 추가:
- **HowTo** (사용 방법)
- **Speakable** (음성 검색 최적화)
- **DefinedTerm** (용어사전 페이지)

## 근거
1. 보고서 §5.2: "LLM이 답변 생성 시 잘 구조화된 데이터를 인용한다"
2. 제로 클릭 환경에서도 **LLM 인용 = 브랜드 노출** → 간접 유입 통로
3. FAQ 섹션 추가만으로 임프레션 8700→35000 / 클릭 4배 케이스 (보고서 §5.2)
4. 장기적으로 "ChatGPT가 추천하는 계산기 사이트" 포지션 확보

## 대안 고려
- **전통 SEO 유지**: 제로 클릭 시대에 구조적 한계
- **AdSense만 극대화**: 트래픽 없으면 수익 0

## 리스크
- LLM 인용 시 링크 클릭률은 여전히 낮을 수 있음 → 브랜드 신뢰 구축이 장기 보상
- 정보 무단 복제 우려 → 우리 계산기 **도구 자체는 복제 불가**. 복제할 수 없는 UX가 해자
