# 검색결과에서 시선을 끄는 제목 링크 작성

*제목 링크*는 웹페이지로 연결되는 Google 검색 및 기타 속성(예: Google 뉴스)의 검색결과 제목입니다. Google에서는 여러 다양한 소스를 사용하여 제목 링크를 자동으로 결정하지만 [시선을 끄는 제목 링크 작성을 위한 권장사항](#page-titles)에 따라 자신이 선호하는 표현을 반영할 수 있습니다.

제목 링크 부분 주위에 강조 표시된 상자가 있는 Google 검색 텍스트 검색 결과 그림


[계란을 바삭하게 튀기는 방법](https://wikipedia.org/wiki/Fried_egg)

## 시선을 끄는 제목 링크 작성을 위한 권장사항

제목 링크는 사용자가 검색결과의 콘텐츠 및 검색어와의 연관성을 빠르게 이해할 수 있도록 도와주므로 매우 중요합니다. 어떤 검색 결과를 클릭할지 결정하는 데 사용되는 주요 정보인 때가 많으므로 웹페이지에 적합한 제목 텍스트를 사용하는 것이 중요합니다.

* **사이트의 모든 페이지에 `<title>` 요소로 지정된 제목이 있는지 확인합니다**.
* `<title>` 요소에 관해 **구체적이고 간결한** 텍스트를 작성합니다. 홈페이지 제목을 '홈'으로 하거나 특정인의 프로필 페이지 제목을 '프로필'로 하는 등의 모호한 설명어는 사용하지 않아야 합니다.
    
  또한 `<title>` 요소에
  포함된 텍스트는 불필요하게 길거나 장황하면지 않아야 합니다.
  `<title>` 요소의 길이에는 제한이 없지만, 일반적으로 기기 너비에 맞추기 위해 필요한 경우에는 Google 검색결과에 제목 링크가 잘려서 표시됩니다.
* **유인 키워드 반복**은 사용하지 않아야 합니다. `<title>` 요소에 몇 가지 구체적인 용어를 사용하는 경우 도움이 되기도 하지만, 같은 단어나 구문을 여러 번 표시할 필요는 없습니다. 'Foobar, foo bar, 푸바, 푸 바'와 같은 제목 텍스트는 사용자에게 도움이 되지 않으며 이렇게 [유인 키워드 반복](https://developers.google.com/search/docs/essentials/spam-policies?hl=ko#keyword-stuffing)을 사용하면 Google과 사용자가 검색결과를 스팸으로 오인할 수 있습니다.
* **`<title>` 요소에 반복되는 텍스트 또는 상용구 텍스트**는 사용하지 않아야 합니다. 사이트의 각 페이지마다 `<title>` 요소에 페이지의 콘텐츠를 설명하는 고유한 텍스트가 있어야 합니다. 예를 들어 상거래 사이트의 페이지마다 '저가 상품 세일'이라는 제목을 지정하면 사용자가 두 페이지 간의 차이점을 구분할 수 없습니다. 일부 정보만 다른 `<title>` 요소의 긴 텍스트('상용구' 제목)도 좋지 않습니다. 예를 들어 '밴드 이름 - 동영상, 가사, 포스터, 앨범, 리뷰, 콘서트 보기'와 같은 텍스트가 있는 모든 페이지의 공통 `<title>` 요소에는 필요 없는 내용이 많이 포함되어 있습니다.

  한 가지 해결책은 `<title>` 요소를 동적으로 업데이트하여 페이지의 실제 콘텐츠를 더욱 효과적으로 반영하는 것입니다. 예를 들어 동영상이나 가사가 포함된 특정 페이지에만 '동영상'과 '가사'와 같은 단어를 포함합니다.
* 간결하게 **제목을 브랜드화**합니다. 사이트 홈페이지의 `<title>` 요소는 사이트에 관한 추가 정보를 포함하기에 적합한 곳입니다. 예:
  > <title>ExampleSocialSite, 만남과 교류의 장소</title>

  그러나 사이트에 있는 모든 페이지의 `<title>` 요소에 이 텍스트를 표시하는 것은
  사이트의 여러 페이지가 같은 검색어로 반환되는 경우 반복적으로 보입니다.
  이 경우 다음과 같이 각 `<title>` 요소의 시작이나 끝에 [사이트 이름](https://developers.google.com/search/docs/appearance/site-names?hl=ko)만 포함하고 나머지 텍스트를 하이픈(-)이나 콜론(:), 막대(|)와 같은 구분자로 분리해 보세요.
  > <title>ExampleSocialSite: 가입하고 새로운 계정을 사용하세요.</title>
* 페이지의 **기본 제목이 명확해야 합니다**. Google은 기본 시각적 제목, 제목 요소, 크고 눈에 띄는 다른 텍스트 등 [제목 링크를 만들 때 여러 가지 소스](#sources)를 살펴봅니다. 시각적 중요도와 가시도가 동일한 제목이 여러 개 있다면 보는 사람이 혼란스러울 수 있습니다. 기본 제목을 페이지에 있는 다른 텍스트와 구분해 표시하고, 페이지에서 가장 눈에 띄게 하세요. 예를 들면 더 큰 글꼴을 사용하거나, 페이지에서 가장 먼저 보이는 `<h1>` 요소에 제목을 포함시키는 방법 등이 있습니다.
* **검색엔진이 페이지를 크롤링하지 못하도록 하려는 경우 주의**하세요.
  사이트에서 [robots.txt](https://developers.google.com/search/reference/robots_txt?hl=ko) 프로토콜을 사용하면 Google에서 페이지를 크롤링하지 못하도록 막을 수 있지만, 여전히 색인이 생성되는 경우도 있습니다.
  예를 들어 Google에서 다른 사람의 사이트에 있는 링크를 통해 페이지를 발견한 경우 페이지의 색인을 생성할 수도 있습니다. Google에서 페이지 콘텐츠에 액세스할 수 없는 경우 페이지 외 콘텐츠를 사용하여 다른 사이트의 앵커 텍스트와 같은 제목 링크를 생성합니다. URL 색인이 생성되지 않도록 하려면 [`noindex` 규칙](https://developers.google.com/search/docs/crawling-indexing/block-indexing?hl=ko)을 사용하면 됩니다.
* 페이지의 **기본 콘텐츠와 동일한 언어와 문자 체계(특정 언어의 문자 또는 알파벳)를 사용**합니다. 예를 들어 페이지를 힌디어로 작성했다면 `<title>` 요소도 힌디어로 작성합니다(제목 텍스트를 영어로 작성하거나 제목을 라틴 문자로 음역하지 않음).
    
  Google은 페이지의 기본 언어 및 문자 체계와 일치하는 제목 링크를 표시하려고 합니다.
  Google에서 `<title>` 요소가 페이지에 있는 기본 콘텐츠의 문자 체계나 언어와 일치하지 않는다고 판단하면 제목 링크로 다른 텍스트를 선택할 수 있습니다.
* **`<title>` 요소에 항공편 가격 정보를 포함하지 않도록 합니다.** Google 시스템에서 항공편 페이지의 제목 링크를 생성할 때 가격 정보가 표시되지 않을 가능성이 높습니다. 항공편 가격이 매우 빠르게(때로는 몇 분마다) 변동하여 제목 링크에 표시되는 가격이 방문 페이지의 실제 가격과 일치하지 않을 수 있기 때문입니다.

## Google 검색에서 제목 링크가 생성되는 방식

Google 검색결과 페이지에서 Google의 제목 링크 생성은 완전히 자동화되어 있으며 페이지 콘텐츠와 웹에 표시되는 페이지 콘텐츠 참조를 모두 고려합니다. 제목 링크의 목표는 각 검색결과를 가장 효과적으로 표현하고 설명하는 것입니다.

Google 검색에서는 다음 소스를 사용하여 제목 링크를 자동으로 결정합니다.

* `<title>` 요소의 콘텐츠
* 페이지에 표시되는 주요 시각적 제목
* heading 요소(예: `<h1>` 요소)
* `og:title` `meta` 태그의 콘텐츠
* 스타일 처리를 통한 크고 눈에 띄는 기타 콘텐츠
* 페이지에 포함된 기타 텍스트
* 페이지의 앵커 텍스트
* 페이지를 가리키는 링크 내 텍스트
* [구조화된 `WebSite` 데이터](https://developers.google.com/search/docs/appearance/site-names?hl=ko#how-to-add-structured-data)

Google에서는 이러한 소스 업데이트를 확인하기 위해 페이지를 다시 크롤링하고 다시 처리해야 하며 이 작업에는 며칠에서 몇 주가 걸릴 수 있습니다. 변경사항이 있는 경우 [Google에 페이지를 다시 크롤링하도록 요청](https://developers.google.com/search/docs/crawling-indexing/ask-google-to-recrawl?hl=ko)할 수 있습니다.

Google에서는 개별 사이트의 제목 링크를 수동으로 변경할 수 없지만 제목 링크가 가능한 한 관련성이 높도록 항상 노력하고 있습니다. [권장사항](#page-titles)을 따라 페이지에 관해 표시되는 제목 링크의 품질을 직접 개선할 수도 있습니다.

## 일반적인 문제 및 Google에서 문제를 관리하는 방법

**검색결과의 제목 링크가 페이지의 `<title>` 요소 또는 기본 제목과 다를 수 있는 이유**: Google에서 [페이지에 문제](#issues)가 있음을 감지하면 앵커나 페이지의 텍스트, 기타 [소스](#sources)를 바탕으로 개선된 제목 링크를 생성하려고 할 수 있습니다.

다음은 검색결과에 표시되는 제목 링크의 가장 일반적인 문제입니다. 이러한 문제를 방지하려면 [시선을 끄는 제목 링크 작성을 위한 권장사항](#page-titles)을 따르세요.

| 일반적인 문제 | |
| --- | --- |
| 절반이 비어 있는 `<title>` 요소 | 제목 텍스트의 일부가 누락된 경우입니다. 예를 들면 다음과 같습니다.  <title>| 사이트 이름</title>  Google 검색에서는 헤더 요소 또는 페이지에서 크고 눈에 띄는 다른 텍스트의 정보를 살펴보고 제목 링크를 생성합니다.  제품 이름 | 사이트 이름 |
| 더 이상 사용되지 않는 `<title>` 요소 | 반복되는 정보에 같은 페이지가 매년 사용되지만 `<title>` 요소는 최신 날짜가 반영되도록 업데이트되지 않은 경우입니다. 예를 들면 다음과 같습니다.  <title>2020년 입학 기준 - 어썸 대학교</title>  이 예에서 페이지에는 '2021년 입학 기준'이라는 크고 눈에 잘 띄는 제목이 있지만 `<title>` 요소는 현재 날짜로 업데이트되지 않았습니다. Google 검색에서는 이러한 불일치를 감지하고 제목 링크에 있는 페이지에 표시되는 제목에서 올바른 날짜를 사용할 수 있습니다.  2021년 입학 기준 - 어썸 대학교 |
| 부정확한 `<title>` 요소 | `<title>` 요소가 페이지 내용을 정확하게 반영하지 못하는 경우입니다. 예를 들어 페이지에 다음과 같은 `<title>` 요소가 포함된 동적 콘텐츠가 있을 수 있습니다.  `<title>`큰 동물 인형, 테디베어, 북극곰 - 사이트 이름</title>  Google 검색에서는 `<title>` 요소가 페이지의 내용을 정확하게 표시하지 않는지 판단하려고 합니다. 페이지 제목이 페이지 콘텐츠를 반영하지 않는다고 판단하면 Google 검색에서는 제목 링크를 수정하여 사용자를 더 효과적으로 지원할 수도 있습니다. 예를 들면 다음과 같습니다.  동물 인형 - 사이트 이름 |
| `<title>` 요소의 마이크로 상용구 텍스트 | 사이트 내 일부 페이지의 `<title>` 요소에 반복되는 상용구 텍스트가 있는 경우입니다. 예를 들어 TV 웹사이트에 시즌 번호가 생략된 동일한 `<title>` 요소를 공유하는 페이지가 여러 개 있어 어떤 페이지가 어떤 시즌을 나타내는지 명확하지 않습니다. 그러면 다음과 같이 `<title>` 요소가 중복되어 생성됩니다.  <title>나의 멋진 TV 프로그램</title>  <title>나의 멋진 TV 프로그램</title>  <title>나의 멋진 TV 프로그램</title>  Google 검색에서는 크고 눈에 띄는 제목 텍스트에 사용된 시즌 번호를 감지하여 제목 링크에 삽입할 수 있습니다.  시즌 1 - 나의 멋진 TV 프로그램  시즌 2 - 나의 멋진 TV 프로그램  시즌 3 - 나의 멋진 TV 프로그램 |
| 명확한 기본 제목이 없음 | 크고 눈에 잘 띄는 제목이 두 개 이상 있으며 이중 어느 텍스트가 페이지의 기본 제목인지 명확하게 알 수 없는 경우입니다. 하나의 페이지에 동일한 스타일 또는 제목 수준을 사용하는 제목이 두 개 이상 있는 경우가 여기에 해당합니다. Google 검색에서 크고 눈에 띄는 제목이 여러 개 있음을 감지하면 이중 첫 번째 제목을 제목 링크의 텍스트로 사용할 수 있습니다. 기본 제목을 페이지에 있는 다른 텍스트와 구분해 표시하고, 페이지에서 가장 눈에 띄게 하세요. 예를 들면 더 큰 글꼴을 사용하거나, 페이지에서 가장 먼저 보이는 `<h1>` 요소에 제목을 넣는 방법 등이 있습니다. |
| `<title>` 요소에 사용된 문자 체계 또는 언어가 일치하지 않음 | `<title>` 요소에 있는 텍스트의 문자 체계나 언어가 페이지의 기본 텍스트 문자 체계 또는 언어와 일치하지 않는 경우. 예를 들어 페이지가 힌디어로 작성되어 있지만 제목에 영어로 작성된 텍스트가 들어 있거나 라틴 문자로 음역된 경우입니다. Google에서 불일치를 감지하면 기본 콘텐츠와 더 일치도가 높은 제목 링크를 생성할 수 있습니다. 문자와 언어가 페이지에서 가장 눈에 띄는 내용과 일치하도록 합니다. |
| `<title>` 요소에 [사이트 이름](https://developers.google.com/search/docs/appearance/site-names?hl=ko)이 중복됨 | 도메인 수준 사이트 이름의 경우 사이트 이름이 [검색결과에 이미 표시되는 사이트 이름](https://developers.google.com/search/docs/appearance/site-names?hl=ko)과 반복되면 Google이 제목 링크에서 사이트 이름을 생략할 수 있습니다. |

## 제목 링크 관련 의견 제출

검색결과에 표시되는 페이지의 제목 링크가 수정되어 있다면 페이지에 Google의 조정 대상이 되는 [문제](#issues)가 있는지 확인하세요. 문제가 없다면 검색결과에 표시되는 제목 링크가 검색어에 더 적합한 것은 아닌지 확인해 보세요.
[Google 검색 센터 도움말 커뮤니티](https://support.google.com/webmasters/community?hl=ko)에 가입하여 페이지 제목 링크에 대해 논의하고 다른 사이트 소유자의 페이지에 관한 의견을 얻으세요.
