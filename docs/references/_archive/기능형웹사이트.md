> 목적: 정보성·도구형 애드센스 사이트를 빠르게 제작·운영하고, 수익 효율을 높이기

|---|--------|------|------|

### 강점
- 세금·정부지원금·금융·법률 등 **고단가(RPM) 키워드** 영역에 집중 → 애드센스 방향 자체는 우수
- 2번(계산기)·4번(ETF)·8번(법률)은 색깔이 뚜렷해 서로 겹치지 않음
- 1차 출처(국세청·한국은행 등) 표기로 신뢰도 확보

### 개선점 / 리스크
- **주제 중복**: 3·5·6·7번(세금·지원금·보험·금융)이 상당 부분 겹침 → 콘텐츠 자기잠식·유사 사이트 위험
  - → 각 사이트 정체성을 뾰족하게 재정의 (예: 하나는 "질문답변", 하나는 "신청방법 가이드")
- **광고 UX**: 3·5번의 팝업/오버레이 광고는 사용자 경험 저해 + 정책 위반 소지 → 배치 점검
- **속도**: 계산기·ETF·데이터 사이트는 JS 로딩 많음 → 광고 스크립트와 겹치면 LCP/CLS 악화

### 기획 방향
- 8개를 따로 키우기보다 **연결 동선 설계**: 도구(계산기) 유입 → 해설(정보) 체류 → 관련 계산기 재순환
- 겹치는 주제를 약점이 아닌 **내부 링크 자산**으로 전환

## 2. 스택별로 도움 되는 GitHub 프로젝트 (4갈래)

### ① Claude Code 생산성 강화 (최우선)
- **hesreallyhim/awesome-claude-code** (~4.9만★) — Claude Code용 스킬·명령어·설정 큐레이션
- **VoltAgent/awesome-claude-code-subagents** (~2.3만★) — 용도별 특화 서브에이전트 100+
- **rohitg00/awesome-claude-code-toolkit** — 에이전트·스킬·명령어·훅·템플릿 종합 툴킷
- ⚠️ 스킬/스크립트는 실행 전 내용 확인 (출처가 제각각)

### ② Cloudflare 배포용 사이트 뼈대(템플릿)
- **ifindev/fullstack-next-cloudflare** (~1.4천★) — Next.js 15 + Workers + D1 + R2 + 인증 (저장·조회 필요한 사이트용)
- **groovy-web/edge-computing-starter** — Hono 기반 경량 엣지 템플릿 (빠름)
- **morrisonak/claude-start-cf** — TanStack Start + Cloudflare + shadcn/ui (Claude Code 시작용, 스택 정확히 일치)
- 💡 **추천**: 콘텐츠 중심 정보성 사이트는 **Astro + Cloudflare Pages** (로딩 빠름 + SEO 유리)
  - 계산기·ETF처럼 상호작용 많은 건 Next/React 계열 유지

### ③ 기능 컴포넌트
- 차트: **ECharts / Chart.js** (시세·지표 그래프)
- 사이트 검색: **Fuse.js**(클라이언트 검색) / **Pagefind**(정적 사이트용, Cloudflare 궁합 좋음)
- 표·데이터: **TanStack Table**

### ④ 품질·수익 최적화 도구
- **chrome-devtools-mcp** (ChromeDevTools 공식) — Claude Code에 MCP 연결
  - 배포 사이트를 Claude가 직접 열어 성능 추적·광고 로딩 영향·콘솔 에러 진단
  - "바이브 코딩 → 배포 → 브라우저 검증 → 수정" 루프 자동화
  - Lighthouse 감사 내장
- **Unlighthouse** — 사이트 전체 일괄 SEO/성능 감사

## 3. 권장 워크플로우 우선순위

1. `awesome-claude-code` 계열로 **바이브 코딩 자체 강화**
2. 새 사이트는 **Astro** 또는 **Cloudflare 전용 템플릿**으로 틀 잡기
3. 기능은 **ECharts · Pagefind** 등 검증된 컴포넌트로 조립
4. **chrome-devtools-mcp**로 성능·광고 품질 상시 점검

## 4. 주의사항 (지켜야 할 선)
- 자동 클릭·트래픽 부풀리기 = 애드센스 무효 트래픽 위반 (계정 정지) → 절대 금지
- 자동화 도구는 **내 사이트 진단·테스트 용도로만**
- 벤치마킹은 "구조·전략 학습"까지만, 글/이미지 복사는 저품질·저작권 문제
