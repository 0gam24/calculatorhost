## 2. URL 전략

### 2-1. 옵션 비교

| 전략 | 예시 | 권장도 | 사유 |
|---|---|---|---|
| 서브디렉토리 | `/en/`, `/ko/` | ★★★★★ | SEO 통합, 운영 단순 |
| 서브도메인 | `en.example.com` | ★★ | DNS·SSL 분리 부담 |
| ccTLD | `example.kr`, `example.com` | ★ | 도메인 비용·관리 비용 ↑ |

**본 표준 권장: 서브디렉토리** (`/ko/`, `/en/`).

### 2-2. 기본 언어 처리

루트(`/`)는 기본 언어로 자동 리다이렉트 또는 기본 언어 콘텐츠 노출:

- (A) `/` → `/ko/` 308 리다이렉트 (URL 일관)
- (B) `/`에 기본 언어(ko) 콘텐츠 직접 노출 (SEO 친화적, 권장)

본 표준은 **(B) 기본 언어를 루트에 노출**.

```
/                 ← 한국어 (기본)
/about            ← 한국어
/en/              ← 영어
/en/about         ← 영어
```

## 3. Astro i18n 설정

### 3-1. astro.config.mjs

```js
import { defineConfig } from 'astro/config';
import sitemap from '@astrojs/sitemap';

export default defineConfig({
  site: 'https://example.com',
  trailingSlash: 'never',
  i18n: {
    defaultLocale: 'ko',
    locales: ['ko', 'en'],
    routing: {
      prefixDefaultLocale: false,  // 기본 언어는 prefix 없음
    },
    fallback: {
      en: 'ko',  // 영어 누락 시 한국어로 폴백
    },
  },
  integrations: [
    sitemap({
      i18n: {
        defaultLocale: 'ko',
        locales: { ko: 'ko-KR', en: 'en-US' },
      },
    }),
  ],
});
```

### 3-2. 디렉토리 구조

```
src/pages/
├── index.astro              → / (한국어)
├── about.astro              → /about (한국어)
├── blog/
│   └── [slug].astro
└── en/
    ├── index.astro          → /en (영어)
    ├── about.astro          → /en/about
    └── blog/
        └── [slug].astro
```

또는 동적 라우팅으로 단일 파일 처리 (Astro 5+ 권장):

```
src/pages/
├── [...locale]/
│   ├── index.astro
│   ├── about.astro
│   └── blog/
│       └── [slug].astro
```

## 4. hreflang (필수)

### 4-1. 모든 다국어 페이지에 상호 hreflang

```html
<!-- /about (한국어) -->
<link rel="alternate" hreflang="ko-KR" href="https://example.com/about" />
<link rel="alternate" hreflang="en-US" href="https://example.com/en/about" />
<link rel="alternate" hreflang="x-default" href="https://example.com/about" />
```

```html
<!-- /en/about (영어) -->
<link rel="alternate" hreflang="ko-KR" href="https://example.com/about" />
<link rel="alternate" hreflang="en-US" href="https://example.com/en/about" />
<link rel="alternate" hreflang="x-default" href="https://example.com/about" />
```

### 4-2. 규칙

- 자기 자신도 포함 (자기참조 hreflang)
- `x-default` 필수 (어느 언어인지 모르는 사용자용)
- 양방향 — A가 B를 참조하면 B도 A를 참조
- 형식: `언어코드-국가코드` (`ko-KR`, `en-US`, `ja-JP`)

### 4-3. sitemap에도 hreflang

```xml
<url>
  <loc>https://example.com/about</loc>
  <xhtml:link rel="alternate" hreflang="ko-KR" href="https://example.com/about"/>
  <xhtml:link rel="alternate" hreflang="en-US" href="https://example.com/en/about"/>
  <xhtml:link rel="alternate" hreflang="x-default" href="https://example.com/about"/>
</url>
```

`@astrojs/sitemap`이 자동 처리.

## 5. 번역 콘텐츠 관리

### 5-1. UI 문자열 — JSON 파일

```
src/i18n/
├── ko.json
└── en.json
```

```json
// src/i18n/ko.json
{
  "nav": {
    "home": "홈",
    "blog": "블로그",
    "about": "소개"
  },
  "buttons": {
    "submit": "제출",
    "cancel": "취소"
  }
}
```

### 5-2. 헬퍼 유틸

```ts
// src/i18n/index.ts
import ko from './ko.json';
import en from './en.json';

const dict = { ko, en } as const;
export type Locale = keyof typeof dict;

export function useTranslations(locale: Locale) {
  return function t(key: string): string {
    const keys = key.split('.');
    let result: any = dict[locale];
    for (const k of keys) {
      result = result?.[k];
    }
    return result || key;
  };
}
```

```astro
---
import { useTranslations } from '../i18n';
const locale = Astro.currentLocale ?? 'ko';
const t = useTranslations(locale);
---
<button>{t('buttons.submit')}</button>
```

### 5-3. 콘텐츠 (블로그 등) — Content Collections

```
src/content/blog/
├── ko/
│   └── article-1.md
└── en/
    └── article-1.md
```

```ts
// src/content/config.ts
import { defineCollection, z } from 'astro:content';

const blog = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    description: z.string(),
    publishedAt: z.date(),
    locale: z.enum(['ko', 'en']),
    translationKey: z.string(),  // 같은 글의 번역 연결
  }),
});

export const collections = { blog };
```

`translationKey`로 같은 글의 다른 언어 버전 매핑 → hreflang 자동 생성.

## 6. 번역 품질

### 6-1. 기계 번역 단독 금지

- ❌ Google 번역만 그대로 사용
- ✅ 기계 번역 + 사람 후편집(MTPE)
- ✅ 전문 번역가 (대규모 사이트, YMYL)

### 6-2. SEO·로컬화

단순 번역이 아니라 **로컬화**:

- 키워드 — 해당 언어권 검색 의도 분석
- 통화·날짜·숫자 — 로케일 포맷
- 단위 — 미터법/제국법
- 문화·관용구 — 직역 금지
- 법적 문구 — 해당 국가 법령에 맞게

### 6-3. 검증

- 원어민 검수
- 같은 길이 가정 금지 (영어→한국어 길이 30% 단축, 독일어 +30%)
- UI 텍스트가 잘리지 않는지 확인

## 7. 로케일 포맷

### 7-1. 날짜·숫자

```ts
new Intl.DateTimeFormat(locale, { dateStyle: 'long' }).format(date);
new Intl.NumberFormat(locale).format(1234567);
new Intl.NumberFormat(locale, { style: 'currency', currency: 'KRW' }).format(29900);
```

### 7-2. 상대 시간

```ts
new Intl.RelativeTimeFormat(locale, { numeric: 'auto' }).format(-3, 'day');
// ko: "3일 전"
// en: "3 days ago"
```

### 7-3. 복수형

```ts
new Intl.PluralRules(locale).select(count);
// 영어: one, other
// 한국어: other (복수 변화 없음)
```

## 8. RTL 언어 지원 (해당 시)

아랍어, 히브리어 등 우→좌 언어 추가 시:

### 8-1. 방향 속성

```html
<html lang="ar" dir="rtl">
```

### 8-2. 논리 속성 사용

```css
/* ❌ 물리적 속성 */
margin-left: 16px;
padding-right: 8px;

/* ✅ 논리적 속성 */
margin-inline-start: 16px;
padding-inline-end: 8px;
```

`Tailwind CSS 4`는 논리 속성 자동 매핑(`ms-4`, `pe-2`).

### 8-3. 아이콘·이미지

방향성 있는 아이콘(화살표 등)은 RTL에서 좌우 반전:

```css
[dir="rtl"] .icon-arrow {
  transform: scaleX(-1);
}
```

## 9. 언어 전환 UI

### 9-1. 위치

- 헤더 우상단 또는 푸터
- 깃발 아이콘만 사용 금지 (국가 ≠ 언어)
- 언어 이름은 **해당 언어로** 표기

```html
<select aria-label="언어 선택">
  <option value="ko">한국어</option>
  <option value="en">English</option>
  <option value="ja">日本語</option>
</select>
```

### 9-2. URL 매핑

언어 전환 시 같은 페이지의 다른 언어 버전으로 이동:

```ts
// /blog/article-1 (한국어) → /en/blog/article-1 (영어)
function switchLocale(currentPath: string, targetLocale: string): string {
  const segments = currentPath.split('/').filter(Boolean);
  if (['ko', 'en'].includes(segments[0])) {
    segments[0] = targetLocale === 'ko' ? '' : targetLocale;
  } else if (targetLocale !== 'ko') {
    segments.unshift(targetLocale);
  }
  return '/' + segments.filter(Boolean).join('/');
}
```

### 9-3. 자동 감지 (선택)

`Accept-Language` 헤더로 첫 방문자 자동 리다이렉트 — **주의**: 검색 엔진 크롤링 방해할 수 있음.

권장: 자동 감지하지 말고 **사용자 선택 유지**(쿠키 또는 URL).
