# 웹사이트 개발 레퍼런스 라이브러리

이 저장소는 **웹사이트를 만들 때 참고하는 문서 라이브러리**다. 코드는 없다.

## 시작점

작업을 시작하기 전에 **반드시** [`docs/references/00-목차-INDEX.md`](docs/references/00-목차-INDEX.md)를 먼저 읽는다.
이 목차가 241개 문서 전체의 진입점이다.

## 문서 우선순위 (충돌 시)

| 순위 | 폴더 | 역할 |
|---|---|---|
| 1 | `docs/references/G-구글-공식가이드/` | Google 관련 **모든 판단의 최종 근거** (156문서) |
| 1 | `docs/references/H-네이버-공식가이드/` | 네이버 관련 **모든 판단의 최종 근거** (55문서) |
| 2 | `docs/references/A~F/` | 실행 방법·프로젝트 규칙 요약 (30문서) |
| 3 | `docs/references/_archive/` | 이력 참고용. **작업 근거로 쓰지 않는다** |

내부 요약(A~F)과 공식 문서(G/H)가 다르면 **항상 G/H가 맞다**. 요약본을 근거로 스펙을 단정하지 말고,
해당 주제의 공식 문서를 열어 확인한 뒤 답한다.

## 검색 규칙

- 특정 스키마·메타태그·정책을 물으면 → G/H에서 **직접 파일을 열어** 확인한다. 기억에 의존하지 않는다.
- Google과 네이버는 규칙이 다르다. 한국 대상 사이트는 **양쪽 다** 확인한다.
  (네이버 고유: `Yeti` 크롤러 · `meta keywords` · RSS 제출 · IndexNow)
- 성능·접근성·보안처럼 공식 문서에 없는 주제는 A~F를 근거로 삼는다.

---

## 실제 프로젝트에 적용하기

웹사이트를 개발하는 프로젝트 폴더의 `CLAUDE.md`에 아래 블록을 붙여넣는다.

```markdown
## 참고 문서 (필수)

웹사이트 관련 작업 전에 아래 레퍼런스를 참조한다.

- 마스터 목차: `D:/Bibe-Code/9999 참고/docs/references/00-목차-INDEX.md`
- Google 공식(1차 출처): `D:/Bibe-Code/9999 참고/docs/references/G-구글-공식가이드/00-목차.md`
- 네이버 공식(1차 출처): `D:/Bibe-Code/9999 참고/docs/references/H-네이버-공식가이드/00-목차.md`

규칙:
1. SEO·구조화데이터·크롤링·메타태그 스펙은 반드시 G/H 공식 문서에서 확인한 뒤 구현한다.
2. 한국 대상 사이트는 Google과 네이버 요구사항을 **모두** 충족시킨다.
3. 성능·접근성·보안·배포는 A~F 요약 문서를 따른다.
4. 배포 전 `E9-출시체크리스트`와 네이버 `06-웹사이트 검증`을 통과시킨다.
```
